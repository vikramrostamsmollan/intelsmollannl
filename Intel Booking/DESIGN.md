# Design System — Intel Training Booking Platform

> **Version:** 1.0 · **Created:** 2026-03-31 · **Status:** Active  
> Generated by `/design-consultation` based on the Sellyo dark redesign + competitive research.

---

## Product Context

- **What this is:** A B2B training booking platform for Intel partner company employees to discover, select, and book Intel-certified training sessions.
- **Who it's for:** Engineers, IT professionals, and sales staff at Dutch Intel partner companies, booking via an Intel partner contract.
- **Space/industry:** B2B enterprise training / event booking — peers: Eventbrite (generic), Luma (consumer), Hopin (enterprise-neutral).
- **Project type:** Multi-step desktop web app — 5-step booking flow + admin/trainer dashboard.
- **Language:** Dutch (nl-NL) — all copy in Dutch.

---

## Aesthetic Direction

- **Direction:** Industrial/Precision Dark — "Precision in the Dark"
- **Decoration level:** Intentional — dot grid overlays on hero sections, radial gradient accent blobs, glassmorphism on nav/modals.
- **Mood:** The UI should feel like a cockpit instrument panel — engineered, high-contrast, purposeful. Not a consumer event app. A professional tool built for people who work in dark IDEs.
- **Competitive positioning:** Every major event/training platform defaults to light mode. Intel Training deliberately breaks this convention. Dark mode signals professionalism and speaks the language of the technical users it serves.
- **Reference design:** [Sellyo.nl](https://sellyo.nl) — dark SaaS aesthetic, glassmorphism, Space Grotesk, ghost borders.

---

## Typography

- **Display/Hero (H1):** **Space Grotesk** 700wt, letter-spacing: -0.05em, line-height: 1.0
  - Rationale: Geometric, technical, distinctive — the opposite of overused Inter. Communicates precision and engineering DNA.
- **H2 / Card titles:** **Space Grotesk** 700wt, -0.03em, 28–32px
- **Body / Subtitles:** **DM Sans** 400wt, line-height: 1.7, color: `#94A3B8`
  - Rationale: Friendly and readable without competing with Space Grotesk's character.
- **UI Labels / Badges:** **Space Grotesk** 700wt, uppercase, letter-spacing: +0.08em, 11–12px
- **Booking references / code:** **DM Mono** 500wt, 14px, Intel Blue tint
- **Loading:** Google Fonts CDN

  ```

## Color

- **Approach:** Balanced — Intel Blue as primary action color, Intel Amber as section/label accent, neutrals for surfaces, semantic colors for status.

### Intel Brand Colors (source: Intel Retail Edge Program materials)

| Token | Hex | Role |
|-------|-----|------|
| Intel Blue | `#0071C5` | Primary CTA, active states, progress indicators |
| Intel Amber | `#F0A500` | Section header labels, callout text, category accents |
| Intel Dark | `#0A0A0F` | Page background |

**Usage rule:** Intel Blue owns all interactive/action elements. Intel Amber is for **labeling and categorization** — section headers (`KLANTEN KRIJGEN:`, `TIJDSCHEMA`), feature category names, and callout highlights. Never use amber on a CTA button.

```

### Dark Mode Strategy

This is a **dark-first** system. No automatic light/dark switching. If a light mode is needed in the future, redesign surfaces (don't invert), reduce color saturation by 10–20%, and maintain the same typographic hierarchy.

## Layout

- **Approach:** Grid-disciplined for the booking flow; creative for landing/hero sections.
- **Grid:** 12 columns, 24px gutters
- **Max content width:** 1200px
- **Booking flow layout:** 2-column (65% form / 35% sticky summary) on desktop, stack on mobile
- **Dashboard layout:** 2-column (65% main / 35% sidebar) on desktop

### Border Radius Scale

```css
--radius-sm:   8px;   /* Inputs, small chips */
--radius-md:  12px;   /* Buttons, inputs */
--radius-lg:  18px;   /* Medium cards */
--radius-xl:  22px;   /* Main cards, panels */

```

**Rule:** Never use border-radius larger than 24px on main layout cards. Pill elements (badges, chips) always use `border-radius: 9999px`.

## Motion

- **Approach:** Intentional — transitions that aid comprehension; no gratuitous animation.

```css
/* Easing */
--ease-out: cubic-bezier(0.0, 0.0, 0.2, 1);
--ease-in:  cubic-bezier(0.4, 0.0, 1, 1);

/* Duration scale */
--dur-micro:  80ms;   /* Icon state changes */
--dur-short:  180ms;  /* Button hover, chip selection */
--dur-med:    300ms;  /* Card state changes, panel opens */
--dur-long:   450ms;  /* Page transitions, modal entrance */
```

**Standard transitions:**
```css
/* All interactive elements */
transition: all 180ms cubic-bezier(0.0, 0.0, 0.2, 1);
```

## Booking Flow — Screen Map

| Step | Screen | Description |
|------|--------|-------------|
| Landing | Event Landing Page | Marketing entry point with hero, features, how-it-works |
| Stap 1 | Training Selection | Card grid with search, filters, category pills |
| Stap 2 | Select Date & Time | Two-column: dark calendar + time slots |
| Stap 3 | Enter Details | Two-column: form + sticky order summary |
| Stap 4 | Bevestiging / Review | Review all details before confirming |
| Stap 5 | Booking Confirmed | Success state with next steps |
| Admin | Event Dashboard | Trainer/admin view with stats, upcoming trainings, activity |

Every booking flow page must have:
1. Glassmorphism nav (fixed top)
2. 5-step progress indicator (steps 1–5, current step active)
3. Content area (dark card on `#0A0A0F` background)
4. Sticky CTA footer (Back + Next/Confirm)

## Do's and Don'ts

### Do
- Use tight letter-spacing on all headlines (-0.03em to -0.05em)
- Use Space Grotesk for all numbers, metrics, and headings
- Layer radial gradient blobs behind hero/landing sections
- Use dot grid overlays on hero sections (not on form/detail pages)
- Show stats with large Space Grotesk numbers + small DM Sans labels
- Make CTAs obvious — one primary blue action per screen
- Use `text-transform: uppercase` + `letter-spacing: +0.08em` on all labels/badges

### Don't
- Don't use white or light backgrounds — always `#0A0A0F`
- Don't use pure black (`#000000`) — use `#0A0A0F`
- Don't use heavy drop shadows — use tonal layering instead
- Don't use Inter, Roboto, or system fonts for headings — always Space Grotesk
- Don't show more than one primary CTA per screen
- Don't use border-radius larger than 24px on main content cards
- Don't use colored borders for layout sectioning — use background shifts
